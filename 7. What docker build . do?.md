## `docker build .` ব্যাখ্যা

`docker build .` কমান্ডটি দিয়ে **Docker Image তৈরি করা হয়**।

### কমান্ডের অংশগুলো:

**`docker build`** → Docker-কে বলছো একটি নতুন image build করতে

**`.`** (dot) → **current directory** কে **build context** হিসেবে ব্যবহার করো, অর্থাৎ এই ফোল্ডারেই `Dockerfile` খোঁজো

---## `docker build .` ব্যাখ্যা

`docker build .` কমান্ডটি দিয়ে **Docker Image তৈরি করা হয়**।

### কমান্ডের অংশগুলো:

**`docker build`** → Docker-কে বলছো একটি নতুন image build করতে

**`.`** (dot) → **current directory** কে **build context** হিসেবে ব্যবহার করো, অর্থাৎ এই ফোল্ডারেই `Dockerfile` খোঁজো

---

### কী হয় ধাপে ধাপে:

1. **Dockerfile খোঁজে** — current directory তে `Dockerfile` আছে কিনা দেখে
2. **Build context পাঠায়** — ঐ folder-এর সব ফাইল Docker daemon-এ পাঠায়
3. **প্রতিটি instruction চালায়** — `FROM`, `RUN`, `COPY`, `CMD` ইত্যাদি লাইন ধরে ধরে execute করে
4. **Layer তৈরি করে** — প্রতিটি step-এ একটি করে layer বানায়
5. **Image তৈরি হয়** — সব শেষে একটি final image ready হয়

---

### সাধারণ উদাহরণ:

```bash
docker build .                  # basic build
docker build -t myapp:1.0 .     # image-এ নাম ও tag দেওয়া
docker build -f MyDockerfile .  # custom Dockerfile নাম দেওয়া
```

---

### মনে রাখো:

| বিষয় | মানে |
|---|---|
| `.` | current folder = build context |
| `-t` | tag/name দেওয়া |
| `-f` | অন্য নামের Dockerfile ব্যবহার |
| Image | build শেষে যা তৈরি হয় |

সহজ কথায় — **`docker build .` মানে "এই ফোল্ডারের Dockerfile দেখে একটা Docker image বানাও"।**

কোনো specific অংশ আরও বিস্তারিত জানতে চাইলে বলো! 😊
