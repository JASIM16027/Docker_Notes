## Docker CLI vs Docker Compose

### Docker CLI — কী এটা?

Docker CLI হলো Docker-এর **direct command-line interface**। এটা দিয়ে তুমি একটা একটা করে container manually চালাও।

**Example — একটা Node.js app চালাতে:**
```bash
# Network তৈরি
docker network create myapp-net

# Database চালাও
docker run -d \
  --name postgres \
  --network myapp-net \
  -e POSTGRES_PASSWORD=secret \
  -v pgdata:/var/lib/postgresql/data \
  postgres:15

# Redis চালাও
docker run -d \
  --name redis \
  --network myapp-net \
  redis:alpine

# App চালাও
docker run -d \
  --name myapp \
  --network myapp-net \
  -p 3000:3000 \
  -e DB_HOST=postgres \
  myapp:latest
```

প্রতিটা service-এর জন্য আলাদা আলাদা command — **অনেক টাইপ করতে হয়, ভুলও হয় বেশি।**

---

### Docker Compose — কী এটা?

Docker Compose হলো **multi-container orchestration tool**। সব কিছু একটা `docker-compose.yml` ফাইলে লিখে রাখো, একটা command-এই সব উঠে যায়।

**Same example — Compose-এ:**
```yaml
# docker-compose.yml
services:
  postgres:
    image: postgres:15
    environment:
      POSTGRES_PASSWORD: secret
    volumes:
      - pgdata:/var/lib/postgresql/data

  redis:
    image: redis:alpine

  myapp:
    build: .
    ports:
      - "3000:3000"
    environment:
      DB_HOST: postgres
    depends_on:
      - postgres
      - redis

volumes:
  pgdata:
```

```bash
docker compose up -d   # সব একসাথে চালু
docker compose down    # সব একসাথে বন্ধ
```

---

### পার্থক্য এক নজরে

| বিষয় | Docker CLI | Docker Compose |
|---|---|---|
| **কতটা container** | একটা করে | অনেকগুলো একসাথে |
| **Config** | Command-এ inline | `yml` ফাইলে |
| **Reusability** | কম | বেশি (ফাইল share করা যায়) |
| **Startup order** | Manual | `depends_on` দিয়ে auto |
| **Network** | Manual তৈরি | Auto তৈরি হয় |
| **Use case** | Quick test, single container | Dev environment, multi-service app |
| **Learning curve** | সহজ | একটু বেশি |

---

### কখন কোনটা ব্যবহার করবে?

**Docker CLI ব্যবহার করো যখন:**
- দ্রুত একটা image test করতে চাও → `docker run nginx`
- Debug করছো, logs দেখছো
- CI/CD-তে single command দরকার
- Production-এ Kubernetes use করছো (Compose দরকার নেই)

**Docker Compose ব্যবহার করো যখন:**
- **Local development** — DB + Backend + Frontend একসাথে
- Team-এ সবাই same environment চাইলে
- Multiple service-এর app (microservices)
- Staging environment

---

### একটা গুরুত্বপূর্ণ কথা

Docker Compose আসলে **Docker CLI-রই wrapper** — পেছনে সে CLI commands-ই চালায়। তাই CLI বোঝা থাকলে Compose অনেক সহজ লাগে।

> **সহজ কথায়:** CLI = হাতে হাতে কাজ। Compose = সব লিখে রাখো, একবারে চালাও। Production-এ বড় apps-এর জন্য Compose অনেক বেশি practical।
